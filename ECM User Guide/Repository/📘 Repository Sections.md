# 🗂️ Repository Sections

## Overview

Repository has **7 navigational sections** in the left sidebar. Each section provides a different view of your files and folders for different workflows.

:::note Quick Reference
Think of sections as **pre-filtered views** of your Repository. They help you find the right files for your current task.
:::

---

## 📊 All 7 Sections at a Glance

| Section | Purpose | Shows | Who Uses |
|---------|---------|-------|----------|
| **Drive** | All accessible files | Everything you have access to | Everyone |
| **My Favorites** | Bookmarked files | Starred files only | Frequent users |
| **My Files** | Incoming shares | Files/folders others shared with you | Collaboration |
| **Shared With Me** | Alternate label | Same as "My Files" in some screens | Collaboration |
| **Shared With Others** | Outgoing shares | Files YOU shared with others | Owners/givers |
| **My Drafts** | Work-in-progress | Unpublished/draft records | Content creators |
| **Trash** | Deleted items | Removable items (30-day retention) | Recovery |

---

## 1️⃣ Drive (Main Storage)

### What is Drive?

**Drive** is the **root/home section** of Repository. It shows all files and folders you have access to, regardless of who created them.

:::info Drive is the Default
When you enter Repository, you always start in Drive unless you select a different section.
:::

### Drive Shows

- ✅ Files you created
- ✅ Files others shared with you (if you have access)
- ✅ Folder structure you can browse
- ✅ Everything in "My Files", "Shared With Me", and more
- ✅ All subfolders and files within

### Using Drive

**Steps:**
1. Click **Sidebar → "Drive"**
2. See all accessible items
3. Browse folders like normal file explorer
4. Navigate to any subfolder

**Use when:**
- ✅ Looking for any file
- ✅ Browsing general collection
- ✅ Want complete overview
- ✅ Organizing files into hierarchy

### Drive Organization Pattern

```
Drive/ (Your visible universe)
├── Projects/
│   ├── ProjectA/
│   │   ├── Contracts/
│   │   ├── Reports/
│   │   └── Communications/
│   └── ProjectB/
├── Finance/
├── HR/
└── Shared-Items/
    ├── Budget 2026/
    └── Sales Materials/
```

**Note:** You see each folder/file ONLY if you have permission to view it

## 2️⃣ My Favorites (Quick Access)

### What are Favorites?

**My Favorites** shows only files/folders you've **starred** (⭐). Think of it as your personal bookmarks.

:::tip Power Feature
Favorites are your quick links to frequently-accessed files without navigating folder trees.
:::

### Adding to Favorites

**Method 1: Star Icon**
1. Hover over file in any section
2. Click **⭐ (Star)** icon
3. Star turns gold
4. File appears in Favorites

**Method 2: Right-Click Menu**
1. Right-click file
2. Click **"Add to Favorites"**
3. File added instantly

### My Favorites Use Cases

| Use Case | Examples |
|----------|----------|
| **Frequently Accessed** | Team charter, budget template, policy docs |
| **Important References** | Company handbook, approval workflows |
| **Current Projects** | Active project folders, draft documents |
| **Shared Items You Use Often** | Files shared with you that you open frequently |

### Organizing Favorites

**Strategy 1: Keep Small (5-15 items)**
- Add only truly frequent files
- Review monthly
- Remove when project ends

**Strategy 2: By Role**
- Admin files
- Meeting templates
- Forms and templates
- Quick reference guides

### Removing from Favorites

**Steps:**
1. Click filled **⭐** icon again, OR
2. Right-click → **"Remove from Favorites"**

**Result:**
- ✅ File stays in Repository
- ✅ Just removed from Favorites section
- ✅ Can re-add anytime

---

## 3️⃣ My Files (Shared With Me)

### What are My Files?

**My Files** shows **only files/folders that other users shared with you**.

:::info Incoming Shares View
My Files = incoming shares from other users. Use this view to quickly access files shared to you.
:::

### My Files Contains

- ✅ Files others shared directly to you
- ✅ Folders others granted you access to
- ✅ Group shares (shared to your team/group)
- ❌ Files you created/uploaded yourself
- ❌ Outgoing shares (files you shared with others)

### Using My Files

**Steps:**
1. Click **Sidebar → "My Files"**
2. See files/folders shared to you by others
3. Open items directly from this view
4. Go to Drive when you need full folder context

### My Files Use Cases

| Use Case | Example |
|----------|---------|
| **Collaboration Inbox** | Teammates shared docs for your review |
| **Assigned Work** | Manager shared files that need action |
| **Quick Access** | Open incoming files without browsing full Drive |
| **Cross-Team Work** | Access folders shared by other departments |

### Organizing My Files

**Common Pattern:**
1. Open "My Files" to see incoming shared items
2. Star important shared files in **My Favorites**
3. Open parent folder in Drive if deeper context is needed
4. Use filters/search to narrow shared items quickly

---

## 4️⃣ Shared With Me (Incoming)

### What is Shared With Me?

**Shared With Me** shows **only files/folders others have shared with you**.

:::note Label Note
In some sidebar configurations, this same view appears as **"My Files"**.
:::

:::info Collaboration Hub
This is where you see all incoming work: shared documents, collaborative projects, assignments.
:::

### Shared With Me Contains

- ✅ Files others shared directly to you
- ✅ Folder access others granted you
- ✅ Group shares (files shared to your group)
- ✅ Items at all permission levels (Viewer to Owner)
- ❌ Your own files (see "My Files")
- ❌ Files in subfolders (only direct shares shown)

### Using Shared With Me

**Steps:**
1. Click **Sidebar → "Shared With Me"**
2. See all files/folders shared TO you
3. Files listed by share date (newest first)
4. Click file to open from this view
5. Can navigate to folder in Drive for context

### Shared With Me Use Cases

| Use Case | Example |
|----------|---------|
| **Collaboration** | Team members share project files |
| **Assignments** | Manager assigns files for review |
| **Review** | Need to review shared documents |
| **Notifications** | See what's new from others |

### Responding to Shared Items

**Workflow:**
1. See file in "Shared With Me"
2. Click file to open/review
3. Edit if you have permission
4. Add to Favorites if frequently used
5. Share with others if needed

---

## 5️⃣ Shared With Others (Outgoing)

### What is Shared With Others?

**Shared With Others** shows **only files/folders you have shared with others**.

:::info Ownership View
This is YOUR perspective: files you're sharing, people you've granted access to.
:::

### Shared With Others Contains

- ✅ Files YOU shared to specific users
- ✅ Folders YOU granted permissions on
- ✅ Group shares (files shared to groups)
- ✅ Public links you created
- ❌ Files others shared to you
- ❌ Shared items you don't own

### Using Shared With Others

**Steps:**
1. Click **Sidebar → "Shared With Others"**
2. See all files/folders you've shared
3. Shows TO WHOM it's shared
4. Can view/modify sharing settings

### Shared With Others Use Cases

| Use Case | Purpose |
|----------|---------|
| **Share Tracking** | What have I shared? With whom? |
| **Permission Review** | Audit your active shares |
| **Revoke Access** | Manage shares you want to end |
| **Update Shares** | Change permission levels on shared items |

### Managing Your Shares

**Common Tasks:**

**Task 1: Review who has a file**
1. See file in "Shared With Others"
2. Right-click → "Manage Permissions"
3. See all users/groups

**Task 2: Revoke Access**
1. Find file in "Shared With Others"
2. Right-click → "Manage Permissions"
3. Remove user/group
4. Click "Save"

**Task 3: Change Permission Level**
1. Find file in section
2. Right-click → "Manage Permissions"
3. Click permission dropdown
4. Select new level

## 6️⃣ My Drafts (Work-in-Progress)

### What are My Drafts?

**My Drafts** shows **only unpublished/draft records**. These are works-in-progress that haven't been finalized.

:::info Draft Status
Drafts are controlled by Content Type (usually File CT). Files in Draft status appear here.
:::

### My Drafts Contains

- ✅ Records you created but didn't publish
- ✅ Files with "Draft" status
- ✅ Incomplete forms/records
- ❌ Published files (even if you created them)
- ❌ Other people's drafts (only yours)

### Using My Drafts

**Steps:**
1. Click **Sidebar → "My Drafts"**
2. See only your unpublished items
3. Continue editing draft records
4. Publish when ready
5. Item moves out of drafts after publication

### My Drafts Workflow

**Typical Process:**
```
1. Create new File Record
   → Status: Draft
   → Appears in "My Drafts"

2. Edit metadata, upload file
   → Still: Draft
   → Still in "My Drafts"

3. Click "Publish" / "Save Final"
   → Status: Published
   → Disappears from "My Drafts"
   → Appears in "My Files" / folder
   → Others can now access (if shared)
```

### My Drafts Use Cases

| Use Case | Purpose |
|----------|---------|
| **Work-in-Progress** | Continue editing half-finished files |
| **Before Publishing** | Review draft before public release |
| **Privacy** | Keep draft private until ready |
| **Batch Publishing** | Review multiple drafts, publish together |

### Publishing from My Drafts

**Steps:**
1. Find draft in "My Drafts" section
2. Click to open
3. Click **"Publish"** or **"Save Final"** button (if visible)
4. Select publish destination (if prompted)
5. Click **"Confirm"**

**Result:**
- ✅ Status changes: Draft → Published
- ✅ Item disappears from "My Drafts"
- ✅ Appears in normal Repository view
- ✅ Others can access (if permissions granted)

## 7️⃣ Trash (Recycle Bin)

### What is Trash?

**Trash** is the **recycle bin** for deleted files and folders. Items stay here for 30 days (configurable) before permanent deletion.

:::info Safety Net
Trash is your safety net. Don't panic if you delete something - it can be recovered within 30 days.
:::

### Trash Contains

- ✅ All deleted files
- ✅ All deleted folders (with contents)
- ✅ Items within 30-day retention window
- ✅ File versions (if versioning enabled)
- ❌ Items older than 30 days (auto-deleted)
- ❌ Items permanently deleted by admin

### Using Trash

**Steps:**
1. Click **Sidebar → "Trash"**
2. See all deleted items
3. Shows deletion date
4. Shows deleted-by user
5. Can recover or permanently delete

### Recovering from Trash

**Steps:**
1. Go to **Trash** section
2. Find item to recover
3. Right-click → **"Restore"**
4. Choose destination (usually original location):
   - Same folder
   - Different folder
   - Root Drive
5. Click **"Restore"** or **"Confirm"**

**Result:**
- ✅ Item restored to chosen location
- ✅ All contents/versions restored
- ✅ Permissions restored
- ✅ Item disappears from Trash

### Permanently Deleting from Trash

**Steps:**
1. Go to **Trash** section
2. Find item
3. Right-click → **"Permanently Delete"**
4. Confirm warning dialog
5. Click **"Permanently Delete"** (confirm)

**Result:**
- ✅ Item removed from Trash
- ✅ Permanently deleted from system
- ✅ Cannot be recovered
- ✅ Space freed up

:::danger Permanent Deletion
Once permanently deleted, cannot be undone. Use with extreme caution.
:::

### Trash Management

**Best Practices:**
- ✅ Empty Trash monthly (free storage)
- ✅ Check Trash before backing up
- ✅ Recover accidental deletions within 30 days
- ❌ Don't rely on Trash for backup
- ❌ Don't assume items deleted after 30 days

### Trash Policies

| Policy | Details |
|--------|---------|
| **Retention Period** | 30 days (configurable by admin) |
| **Auto-Delete** | After 30 days, removed automatically |
| **Manual Empty** | Right-click Trash → "Empty Trash" |
| **Storage Impact** | Trash items count toward quota |

## 🎯 Section Navigation Tips

### Switching Between Sections

**Quick Navigation:**
1. Click section name in sidebar
2. View changes instantly
3. Toolbar/search remain same
4. Can still use search across sections

### Breadcrumb Navigation

**Current Location Shown:**
```
Repository > Drive > Finance > 2026 > Q1
OR
Repository > My Files
OR
Repository > Shared With Me
```

Click any breadcrumb to jump back

### Cross-Section Workflow

**Example: Approval Process**
1. Start: **"Shared With Me"** → See assignment from manager
2. Open file and review
3. Switch to **"My Files"** → Add your notes/version
4. Switch to **"Shared With Others"** → Share updated version with approver
5. Return to **"Shared With Me"** → See approver's response

## 📊 When to Use Each Section

### Section Selection Guide

| Goal | Use This Section |
|------|------------------|
| Find ANY file | **Drive** |
| Find MY file | **My Files** |
| Access frequent files | **My Favorites** |
| See incoming work | **Shared With Me** |
| Check what I shared | **Shared With Others** |
| Find unfinished work | **My Drafts** |
| Recover deleted item | **Trash** |
| View org structure | **Drive** → Browse folders |
| Manage team files | **Drive** → Folder location |

## 💡 Pro Tips

### Tip 1: Create Sub-Favorites
Within "My Favorites", you can:
- Create a "Frequent" folder
- Add shortcuts to 3-5 most-used files
- Reduces searching time

### Tip 2: Use My Drafts for QA
Before publishing:
1. Create in "My Drafts"
2. Self-review
3. Peer-review (share with reviewer)
4. Publish when approved

### Tip 3: Monitor "Shared With Me"
1. Check regularly (like email inbox)
2. Prioritize new shares from manager
3. Archive/move completed items
4. Keep section clean

### Tip 4: Empty Trash Monthly
1. Go to Trash
2. Files older than 30 days
3. Permanently delete old items
4. Free up storage quota

### Tip 5: Use Drive for Teaching
Onboard new users:
1. Start with Drive
2. Show folder hierarchy
3. Explain permissions
4. Then introduce other sections

## 🆘 Common Issues

### Issue: Can't see file in My Files

**Cause:** You didn't create it
**Solution:** Check "Shared With Me" instead

### Issue: Folder moved to wrong section

**Cause:** Sections filter by criteria
**Solution:** Use Drive to browse actual location

### Issue: Too many items in Shared With Me

**Cause:** Haven't organized received items
**Solution:** 
- Archive/delete old shares
- Move frequently-used to Favorites
- Add to Drive folder for organization

### Issue: Recovery deadline missed (>30 days)

**Cause:** Item outside retention window
**Solution:** Contact admin for recovery options
