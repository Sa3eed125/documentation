# Workflow - Complete Connectors & Integration Reference

This guide documents all available connectors in Contellect Workflow for integrating with external systems.

---

## Overview

Connectors enable workflows to communicate with external systems and services. When creating or editing a workflow task, you can select from various pre-built connectors by going to **Change element** → **Connectors** section.

---

## Available Connectors

### 1. PostgreSQL Database Connector

**Icon**: Database icon (🗄️)  
**Category**: Database Integration  
**Purpose**: Execute database operations on PostgreSQL databases  
**When to Use**:
- Query database records
- Insert/update/delete data
- Execute stored procedures
- Fetch data for validation
- Log operations to database

**Configuration Sections**:

#### General Section
- **Name**: Connector instance name
- **Description**: What this connector does
- **Connector Type**: PostgreSQL Database Connector (fixed)

#### Template Section
- **Database Connection**:
  - Server/Host
  - Port (default: 5432)
  - Database Name
  - Username
  - Password (encrypted)
  - Connection Pool Size
  - Connection Timeout (seconds)

- **Query Type**:
  - SELECT - Retrieve data
  - INSERT - Add new records
  - UPDATE - Modify existing records
  - DELETE - Remove records
  - EXECUTE - Run stored procedure
  - BATCH - Multiple operations

#### Operation Section
- **SQL Query**: SQL statement to execute
- **Query Parameters**: Variable mappings (?)
- **Timeout**: Query execution timeout
- **Max Rows**: Result limit (for SELECT)
- **Fetch Mode**: Single row or multiple rows

#### Authentication Section
- **Connection Method**:
  - Direct connection
  - Connection string
  - Connection pool
- **Credentials**:
  - Username/Password
  - SSL/TLS encryption
  - Certificate-based auth

#### Input Mapping Section
- **Parameters**: Map workflow variables to SQL parameters
- **Example**:
  ```
  SQL: SELECT * FROM invoices WHERE amount > ?
  Parameter 1: ${minAmount}
  ```

#### Output Mapping Section
- **Result Variables**: Map query results to workflow variables
- **Examples**:
  - queryResult → invoiceList
  - rowCount → totalRecords
  - lastInsertId → newRecordId

#### Error Handling Section
- **On Connection Error**:
  - Retry with backoff
  - Fail immediately
  - Skip operation
- **On Query Error**:
  - Use default value
  - Log and continue
  - Escalate to manager
- **Retry Count**: 0-5 retries
- **Retry Delay**: Seconds between retries

#### Execution Listeners
- **Before Query**: Pre-execution script
- **After Query**: Post-execution script
- **On Error**: Error handling script

#### Example Data
- Sample query: `SELECT * FROM invoices LIMIT 1`
- Sample result: JSON structure of return data

### 2. SMTP Email Connector

**Icon**: Email icon (📧)  
**Category**: Communication/Notification  
**Purpose**: Send emails using SMTP protocol  
**When to Use**:
- Send notifications
- Alert users
- Send approvals/rejections
- Generate and send reports
- Send confirmations

**Configuration Sections**:

#### General Section
- **Name**: SMTP Email connector instance
- **Description**: Email sending purpose
- **Connector Type**: SMTP Email Connector (fixed)

#### Template Section
- **SMTP Server Settings**:
  - Server Address (smtp.gmail.com, etc.)
  - Port (25, 465, 587)
  - Encryption: None, SSL/TLS, STARTTLS
  - Username
  - Password (encrypted)
  - From Address (sender email)
  - From Display Name

- **Email Template**:
  - Subject Line
  - Email Body (plain text or HTML)
  - Template Variables: `{{recipientName}}`, `{{taskTitle}}`, etc.
  - Include Attachments: Yes/No

#### Operation Section
- **Recipients**:
  - To Address(es)
  - CC Address(es)
  - BCC Address(es)
  - Dynamic recipients from variable
  - Recipient list from database

- **Message Content**:
  - Subject: Static or dynamic
  - Body: HTML or plain text
  - Attachments: File paths
  - Inline Images: Yes/No

- **Priority**: Normal, High, Low
- **Read Receipt**: Requested or not
- **Send Retries**: 0-3 retries

#### Authentication Section
- **SMTP Authentication**:
  - Username/Password
  - OAuth2 token (for Gmail, Office 365)
  - Certificate-based
  - API Key
- **TLS Certificate**: Validation required

#### Input Mapping Section
- **Template Variables**:
  - `{recipientEmail}` → Workflow variable
  - `{recipientName}` → Workflow variable
  - `{approvalStatus}` → Workflow variable
  - `{taskDeadline}` → Workflow variable
  - `{amountTApprove}` → Workflow variable

#### Output Mapping Section
- **Result Variables**:
  - emailSent → true/false
  - messageId → email ID
  - deliveryStatus → Delivered/Failed/Pending
  - timestamp → Send time

#### Error Handling Section
- **On Send Failure**:
  - Retry after delay
  - Log error and continue
  - Escalate to admin
  - Queue for later retry
- **Invalid Email Address**: Skip or fail
- **SMTP Connection Error**: Retry or alert

#### Execution Listeners
- **Before Send**: Validate recipients
- **After Send**: Log delivery
- **On Error**: Alert admin

#### Example Data
```
From: workflow@contellect.com
To: manager@company.com
Subject: Invoice Approval Required: {{invoiceId}}
Body:
Please review and approve invoice #{{invoiceId}}
Amount: ${{amount}}
Due: {{dueDate}}
Approve: {{approvalLink}}
```

**Advanced Features**:
- **Email Templates**: Pre-designed templates library
- **Bulk Send**: Send to multiple recipients
- **Scheduled Send**: Send at specific time
- **Track Opens**: Email read tracking
- **Click Tracking**: Link tracking

- Exchange files with external systems

**Configuration Sections**:

#### General Section
- **Name**: SFTP connector instance
- **Description**: File transfer purpose
- **Connector Type**: SFTP File Connector (fixed)

#### Template Section
- **Server Configuration**:
  - Server Hostname/IP
  - Port (default: 22)
  - Username
  - Authentication Type:
    - Password-based
    - Private Key (SSH key)
    - Public Key authentication
  - Private Key Path (if key-based)
  - Passphrase (for encrypted keys)

- **Connection Settings**:
  - Connection Timeout: Seconds
  - Keep-Alive: Yes/No
  - Connection Pool: Yes/No
  - Max Connections: 1-10

#### Operation Section
- **File Operation Type**:
  - Upload
  - Download
  - List files
  - Delete
  - Rename
  - Create folder

- **Upload Options**:
  - Local file path
  - Remote destination path
  - Overwrite existing: Yes/No
  - Preserve file permissions
  - Create remote folders

- **Download Options**:
  - Remote file path
  - Local destination path
  - Overwrite local: Yes/No

- **List Options**:
  - Remote directory path
  - File pattern filter (*.pdf, invoice_*.xml)
  - Recursive listing
  - Max results

#### Authentication Section
- **SSH Key Management**:
  - Private key storage (encrypted)
  - Key passphrase (encrypted)
  - Known hosts verification
  - Strict host key checking

#### Input Mapping Section
- **File Path Variables**:
  - localFilePath → Workflow variable
  - remoteFilePath → Workflow variable
  - filePattern → Workflow variable
  - directory → Workflow variable

#### Output Mapping Section
- **Result Variables**:
  - fileTransferred → true/false
  - fileName → transferred file name
  - fileSize → bytes
  - timestamp → transfer time
  - fileList → array of files
  - fileCount → number of files

#### Error Handling Section
- **On Connection Error**:
  - Retry connection
  - Fail immediately
  - Use fallback server
- **On File Not Found**: Skip or alert
- **On Permission Denied**: Alert admin
- **On Transfer Incomplete**: Retry or archive partially

#### Execution Listeners
- **Before Transfer**: Validate file/path
- **After Transfer**: Verify checksum
- **On Error**: Log and notify

#### Example Data
```
Upload:
  Local: /tmp/report_{{taskId}}.pdf
  Remote: /uploads/reports/{{year}}/{{month}}/

Download:
  Remote: /exports/invoice_*.xml
  Local: /data/imports/

List:
  Remote: /incoming/
  Pattern: *.csv
  Result: [file1.csv, file2.csv]
```

**Advanced Features**:
- **Batch Operations**: Transfer multiple files
- **Compression**: Zip before upload
- **Encryption**: Encrypt transferred files
- **Checksums**: Verify file integrity
- **Scheduling**: Scheduled transfers

## Error Handling & Recovery

Every connector has built-in error handling:

```
Execution Flow:
1. Validate Connection
   ├─ If failed → Retry or Escalate
   └─ If success → Continue
   
2. Prepare Operation
   ├─ If invalid → Log error, use default
   └─ If valid → Execute
   
3. Execute Operation
   ├─ If failed → Check error type
   ├─ If timeout → Retry or abandon
   └─ If success → Map results
   
4. Handle Results
   ├─ Validate output
   ├─ Map to variables
   └─ Continue workflow
```

## Security Considerations

- **Credentials**: Store encrypted, never in logs
- **Data Transit**: Use SSL/TLS for all connections
- **Access Control**: Limit connector permissions
- **Audit Trail**: Log all connector operations
- **Sensitive Data**: Mask in error messages
- **Updates**: Keep connectors patched
- **Testing**: Test in non-production first
